## Session 2

### Topics Covered

* Arithmetic Expressions
* Integer Division
* Modulo Operator
* Type Conversion
* Type Casting
* Math Class

### Key Learnings

* Integer division truncates fractional values.
* Modulo represents the remaining value after complete grouping.
* Type conversion can occur automatically.
* Type casting is an explicit instruction to convert a value.

### Analogies Created

* Modulo is similar to grouping 10 people into groups of 3 and observing the remaining person.
* Type casting is explicitly instructing Java how to interpret a value.

### New Concepts

* Scientific notation using exponential representation (e.g., 12.34e-6).

## Session 3

### Topics Covered

* Debugging Interlude
* Compile-Time Errors
* Runtime Errors
* Silent Logical Errors
* Debugging Techniques

### Key Learnings

* Syntax correctness does not guarantee logical correctness.
* Execution success does not guarantee requirement satisfaction.
* Displaying intermediate values helps isolate defects.
* Debugging often requires tracing program state step-by-step.

### Strong Insight

A program can:

* Compile successfully
* Execute successfully
* Produce output successfully

and still fail to satisfy the actual requirement.

This insight was reinforced through a previously encountered embeddings-related bug where multiple embeddings were generated but only one embedding was returned.

### Key Learnings

### Analogies Created

* Individual Human (Maheshwaran) → Object

This analogy helped reinforce the distinction between a general definition and a specific instance.

# Strongest Insight Of The Day

A running program is only the beginning of validation, not the end.

Software correctness exists at multiple layers:

* Syntax Correctness
* Logic Correctness
* Requirement Correctness

Success at one layer does not imply success at the next.
