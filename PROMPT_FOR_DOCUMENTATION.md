# Prompt For Documentation

At the end of a learning day, use the following prompt.

---

You are helping document my learning journey.

Generate a SUMMARY.md for today's learning.

Important Rules

1. Use only concepts that were actually discussed.
2. Use only doubts that were actually asked.
3. Use only questions that were actually discussed.
4. Do not invent information.
5. Do not invent concepts.
6. Do not invent doubts.
7. Do not invent AI questions.
8. Do not create artificial insights.
9. Summarize understanding rather than copying lesson text.
10. Focus on future revision value.
11. Distinguish between:

* Concepts learned
* Learning artifacts
* Engineering intuition developed

12. Do not list course example programs as implemented unless they were actually implemented, modified, or written by the learner.
13. Debugging discussions, transcript discussions, reasoning exercises, and AI verification discussions may be documented as learning artifacts.

Generate the following sections:

# Day XX Summary

## Date

## Learning Context

## Concepts Learned

For each concept:

* Concept Name
* Session Reference
* What I Learned
* Related Artifacts
* Key Understanding

## Doubts Asked To AI

For each doubt:

* My Question
* AI Explanation
* Final Understanding

## Questions AI Asked Me

For each question:

* AI Question
* My Answer
* My Reasoning
* Final Outcome
* Correct Understanding

## Learning Artifacts

Document anything that contributed meaningful learning value.

Possible artifact types:

* Program
* Debugging Case
* Discussion
* Transcript Discussion
* Exercise
* AI Verification
* Other

For each artifact:

* Artifact Type
* Title/File Name
* Purpose
* Concepts Used

## Engineering Intuition Developed

For each intuition:

* Insight
* Why It Matters

Only include intuitions that were explicitly discussed or derived during learning.

Do not invent insights.

## Learning Insights

* Most Important Concept Today
* Most Valuable Doubt
* Most Valuable AI Question
* Concepts Requiring Revision

## End of Day Reflection

## Navigation

* Concepts
* Learning Artifacts
* Doubts Discussed
* AI Verification Questions

Before generating the summary:

* Cross-check concepts, doubts, AI questions, artifacts, intuitions, and reflections against the actual discussion.
* If information is missing, leave the section empty rather than inventing content.
* The output should act as both a revision guide and a learning archive.
* Prefer accuracy over completeness.
* Distinguish between:

  * What was learned
  * What was implemented
  * What was discussed
* Do not assume a program exists simply because it appeared in course material.

# Workflow Notes

I use a separate summary template to maintain documentation consistency.

Before using this prompt:

1. Copy and paste this prompt.
2. Provide the contents of SUMMARY_TEMPLATE.md.
3. Ask the AI to generate the day's summary by following the template.

The template structure is only a formatting reference.

A learning day may contain:

* Any number of learning sessions.
* Any number of concepts.
* Any number of doubts.
* Any number of AI verification questions.
* Any number of learning artifacts.
* Any number of engineering intuitions.

The generated summary should adapt to the actual learning content rather than assume a fixed structure.

Template Reference:

* SUMMARY_TEMPLATE.md

# Human Validation Required

Before saving the generated summary:

* Verify factual accuracy.
* Verify concepts.
* Verify doubts.
* Verify AI questions.
* Verify explanations.
* Verify reflections.
* Verify artifact references.
* Verify navigation links.

The learner is responsible for final approval.

AI assists documentation.

The learner validates documentation.
